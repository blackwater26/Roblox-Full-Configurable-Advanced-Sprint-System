# Contributing Guidelines

Thank you for your interest in contributing to this project!
Contributions are welcome and appreciated. Please follow the guidelines
below to ensure a clean and consistent development process.

## How to Contribute

1. Fork this repository.
2. Create a new branch for your changes:
   `git checkout -b feature/your-feature-name`
3. Make your changes and commit them with a clear message:
   `git commit -m "Add new feature"`
4. Push your branch to your fork:
   `git push origin feature/your-feature-name`
5. Open a Pull Request against the `main` branch.

## Contribution Rules

- Follow the existing **modular and event-driven architecture**.
- Avoid tightly coupling new logic to existing systems.
- Add clear and concise comments for any new or non-trivial logic.
- For large changes or architectural updates, please open an **Issue**
  first to discuss the proposal before submitting a Pull Request.

## Issues & Feedback

Bug reports, suggestions, and improvement ideas are always welcome.
Feel free to open an Issue to start a discussion.
